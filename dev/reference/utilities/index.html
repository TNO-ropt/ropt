    scale_variables(config, variables, axis)

    Scale variables.

    Given an ensemble optimizer configuration object and a vector of variables, this function applies an offset and scale to the vector values.

    As the variables input may be a multi-dimensional array, the index of the axis that designates the variables should be provided through the axis argument.

    The scaled variables.

    axis int

    The variables axis.

    		 required

    Returns:

    Type Description

    The scaled variables, or None if no scaling is applied.

    scale_back_variables(config, variables, axis, *, correct_offsets=True)

    Scale back variables.

    Given an ensemble optimizer configuration object and a vector of scaled variables, scale their values back to the original range. Normally this includes correcting for offsets, but if a difference value is being rescaled, the correct_offsets flag can be used to disable this.

    As the variables input may be a multi-dimensional array, the index of the axis that designates the variables should be provided through the axis argument.

    Parameters:

    Name Type Description Default
    config EnOptConfig

    The ensemble optimizer configuration object.

    		 required
    variables NDArray[float64]

    The scaled variables.

    		 required
    axis int

    The variables axis.

    		 required
    correct_offsets bool

    If True also correct for offsets.

    		 True

    Returns:

    Type Description
    NDArray[float64] | None

    The unscaled variables, or None if no scaling is applied.

    scale_objectives(config, objectives, scales, axis)

    Scale objective function values.

    Given an ensemble optimizer configuration object, this function scales the provided objective values. It divides them by the scale values given in the configuration object (if not None), and optionally also by the values given in the scales argument.

    As the objectives input may be a multi-dimensional array, the index of the axis that designates the objectives should be provided through the axis argument.

    		 required

    The objectives axis.

    		 required

    Returns:

    Type Description
    NDArray[float64] | None

    The scaled objectives or None if no scaling was applied.

    format_tuple(values, delimiters=':')

    Format a tuple of strings into a single string.

    This function converts a tuple into a single string by joining the tuple items with specified delimiters. The delimiters argument can contain multiple characters, each of which will be used in turn to separate the items in name. If fewer delimiters are provided than needed, the final delimiter is reused for any remaining items. If delimiters is not specified, : is used by default.

    Parameters:

    Name Type Description Default
    values str | tuple[Any, ...]

    The tuple of strings to format.

    		 required
    delimiters str

    The delimiters to use, applied sequentially.

    		 ':'

    Returns:

    Type Description
    str

    The formatted string.

    ImmutableBaseModel

    Bases: BaseModel

    Base model for immutable classes.

    This model serves as an alternative to frozen Pydantic classes. It is particularly useful when post-initialization validators are required, as these validators may not function properly with frozen Pydantic classes.

    ropt.config.validated_types

    Annotated types for use with Pydantic models.

    These types can be used to convert input values to a desired type and guarantee certain properties. They include types that convert inputs to immutable NumPy arrays of specified dimension and type:

    • Array1D: For converting sequences to immutable one-dimensional floating-point arrays.
    • Array2D: For converting sequences to immutable two-dimensional floating-point arrays.
    • ArrayIndices: For converting sequences to immutable arrays containing indices of any dimension.
    • ArrayEnum: For converting sequences to values of numerical enumerations of any dimension.
    • Array1DInt: For converting sequences to immutable one-dimensional integer arrays.
    • Array1DBool: For converting sequences to immutable one-dimensional boolean arrays.

    Additionally, the following convenience types create sets or tuples, ensuring that single values are embedded in a set or tuple, respectively:

    • ItemOrSet[T]: Create a set of type T.
    • ItemOrTuple[T]: Create a tuple of type T.

    Array1D = Annotated[NDArray[np.float64], BeforeValidator(_convert_1d_array)] module-attribute

    Convert to an immutable 1D numpy array of floating point values.

    Array2D = Annotated[NDArray[np.float64], BeforeValidator(_convert_2d_array)] module-attribute

    Convert to an immutable 2D numpy array of floating point values.

    ArrayIndices = Annotated[NDArray[np.intc], BeforeValidator(_convert_indices)] module-attribute

    Convert to an immutable numpy array of indices.

    ArrayEnum = Annotated[NDArray[np.ubyte], BeforeValidator(_convert_enum_array)] module-attribute

    Convert to an immutable numpy array of numerical enumeration values.

    Array1DInt = Annotated[NDArray[np.intc], BeforeValidator(_convert_1d_array_intc)] module-attribute

    Convert to an immutable 1D numpy array of integer values.

    Array1DBool = Annotated[NDArray[np.bool_], BeforeValidator(_convert_1d_array_bool)] module-attribute

    Convert to an immutable 1D numpy array of boolean values.

    UniqueNames = Annotated[tuple[Any, ...], AfterValidator(_check_duplicates)] module-attribute

    Check for duplicates in a tuple, raising ValueError if duplicates are found.

    ItemOrSet = Annotated[set[T], BeforeValidator(_convert_set)] module-attribute

    Convert to single value to a set containing that value, passes sets unchanged.

    ItemOrTuple = Annotated[tuple[T, ...], BeforeValidator(_convert_tuple)] module-attribute

    Convert to single value to a tuple containing that value, passes sets unchanged.
